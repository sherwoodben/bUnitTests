Detailed Description

a struct which represents a unit test

Note
this is intentionally "empty" (aside from a custom ctor) because the way the unit testing framework implements tests is by inheriting from this (very basic) struct for each test to be implemented. The ctors of the specific tests provide the arguments for this ctor, so the macro to create each individual test can (indirectly) provide a name and a function to this ctor and everything plays nicely
